# Avalonia Port Instructions

This project is a WPF → Avalonia UI port of CiderPress2, an Apple II
disk image and file archive utility.

## Porting Guidance

All porting documents are in `cp2_avalonia/guidance/`. Before starting
any iteration, read these in order:

1. `guidance/PORTING_OVERVIEW.md` — architecture, project structure,
   naming conventions, and cross-cutting concerns
2. `guidance/Pre-Iteration-Notes.md` — setup requirements and
   foundational decisions
3. The relevant `guidance/Iteration_N_Blueprint.md` for the iteration
   you are executing

## Key Conventions

- **No MVVM framework / No ReactiveUI** — use code-behind + controller
  pattern, matching the WPF architecture
- **Avalonia 11.x** with Fluent theme
- **Target framework:** `net8.0` (not `net8.0-windows`, not `net9.0`)
- **RelayCommand** with dual-constructor (Action execute, Func<bool>
  canExecute) for command wiring
- **INotifyPropertyChanged** for data binding (no ReactiveObject)
- **avares://** URI scheme for resources
- **DataGrid** from `Avalonia.Controls.DataGrid` NuGet package

## Execution Rules

- Follow blueprints step-by-step. Complete and verify each step before
  proceeding to the next.
- Do not skip steps or combine them unless the blueprint explicitly
  says to.
- If a step is ambiguous, refer back to the overview and pre-iteration
  notes for context.
- Do not add features, refactor code, or make improvements beyond what
  the blueprint prescribes.
- The WPF source in `cp2_wpf/` is the reference implementation — when
  in doubt, check how WPF does it.
